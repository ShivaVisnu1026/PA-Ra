## 技術指標類

### 範例：CustomEMA（自訂指數移動平均）
> **狀態**：📝 此為範例，實際使用時請刪除

- **檔案**：`scripts/functions/production/CustomEMA.xs`
- **用途**：計算自訂的指數移動平均線

**完整說明**：
此函數提供更靈活的 EMA 計算方式，支援自訂平滑係數，可用於開發特殊的均線系統。

**參數**：
- `Price` (numericseries): 輸入價格序列，通常使用 Close
- `Length` (numericsimple): 計算週期，必須 > 0
- `SmoothFactor` (numericsimple): 平滑係數，範圍 0~1，預設 2/(Length+1)

**返回值**：
- (numeric): EMA 數值

**使用範例**：
```xscript
// 基本使用
vars: MyEMA(0);
MyEMA = CustomEMA(Close, 20, Default);

// 自訂平滑係數
vars: FastEMA(0);
FastEMA = CustomEMA(Close, 20, 0.15);  // 比標準 EMA 更快回應

// 應用在其他價格
vars: HighEMA(0);
HighEMA = CustomEMA(High, 10, Default);
```

**注意事項**：
- Length 必須 > 0，否則返回 0
- SmoothFactor 建議在 0.05 ~ 0.3 之間
- 初始值使用 SMA 計算

**相關函數**：
- 內建 `Average()` - 簡單移動平均
- 內建 `XAverage()` - 指數移動平均

**版本記錄**：
- v1.0 (2024-01-01): 初始版本
- v1.1 (2024-02-01): 新增 SmoothFactor 參數

## 📝 如何新增函數

### 步驟 1：開發函數
在 `scripts/functions/brainstorm/` 開發和測試你的函數。

### 步驟 2：測試驗證
確保函數在各種情況下都能正確運作：
- 正常參數
- 邊界值（最大/最小）
- 異常值（0、負數、空值）
- 不同資料序列

### 步驟 3：移到 production
測試完成後，將函數檔案移到 `scripts/functions/production/`。

### 步驟 4：登記索引
在本檔案登記函數資訊，使用上方的格式。

### 步驟 5：提供範例
在 `examples/` 資料夾提供使用範例（可選但建議）。

## 💡 最佳實踐

### 命名規範
- 使用清晰的英文名稱
- 大駝峰命名法（PascalCase）
- 避免使用保留字
- 名稱要表達函數用途

### 參數設計
- 參數順序：必要參數在前，可選參數在後
- 使用合理的預設值
- 參數數量不宜過多（建議 ≤ 5 個）

### 效能考量
- 避免不必要的重複計算
- 考慮函數的時間複雜度
- 大量資料處理要特別注意效能

### 文檔撰寫
- 說明要清晰完整
- 範例要實用且可直接運行
- 注意事項要明確
