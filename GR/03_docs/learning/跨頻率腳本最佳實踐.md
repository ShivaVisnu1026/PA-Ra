## 跨頻率腳本最佳實踐（xf_ 函數）

更新日期：2024-12-19

### 核心觀念
- 在較短頻率（如日、分）中引用較長頻率（如週、月）資料時，請使用 `xf_` 函數族。
- `xf_XAverage`, `xf_RSI`, `xf_EMA`… 會處理跨頻率對齊與期初化問題。
- 自訂跨頻率函數通常需要 `xf_getdtvalue`, `xf_GetCurrentBar` 來偵測換期與期序。

參考：
- 官方函數：xf_XAverage（跨頻率 EMA）
- 教學：如何使用跨頻率技術指標建構策略雷達

### 常用範例

- 週 EMA20 在日腳本中計算：
```xs
wk_close      = GetField("Close", "W");
wk_ema20      = xf_XAverage("W", wk_close, 20);
// 取上一週 EMA：來源序列整體往前一週再計算，而非用 wk_ema20[1]
wk_ema20_prev = xf_XAverage("W", wk_close[1], 20);
```

- 週 RSI < 20 且日 KD 黃金交叉（示意）：
```xs
stochastic(9, 3, 3, rsv_d, kk_d, dd_d);
if xf_RSI("W", Close, 5) <= 20 and kk_d crosses over dd_d then ret = 1;
```

### 反例與陷阱
- `wk_ema20_prev = wk_ema20[1]`：這只是在主頻率上往前一天，非上一週。
- 在跨頻率腳本內加 `if barfreq <> "Day" then raiseruntimeerror(...)`：會阻擋掃描/回測，改以 `xf_` 函數顯式指定頻率。

### 設計建議
- 清楚分層：短頻率觸發訊號 + 長頻率趨勢濾網。
- Debug：同時輸出主值、前值、差值三欄（或以 `RetMsg` 顯示）以核對圖面。
- 排序：選股腳本同時只指定一個 `order :=` 欄位，避免編譯警告。

### Pin Bar 判定簡化（日頻率）
- 以全 K 範圍為基準：
  - 長影線占比 ≥ 60%
  - 反側影線占比 ≤ 15%
  - 實體占比 ≤ 35%
  - 實體高點位於低點 + 35% 區間內

> 門檻應依市場調性微調，避免過嚴導致漏抓。
