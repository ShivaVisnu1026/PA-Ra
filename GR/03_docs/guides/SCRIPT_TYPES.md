# XScript 五種腳本類型完整說明

> 本文檔詳細說明 XScript 的五種腳本類型，包括用途、限制、資料存取權限和最佳實踐

---

## 📋 類型總覽

| 類型 | 英文名稱 | 主要用途 | 執行環境 | 資料存取 |
|------|---------|---------|---------|----------|
| 🎯 指標 | Indicator | 計算技術指標並視覺化 | 圖表上 | 價量資料 |
| 🔔 警示 | Alert | 條件觸發時通知使用者 | 即時監控 | 價量資料、財務數據 |
| 🔍 選股 | Screener | 篩選符合條件的股票 | 批次掃描 | 價量、財務、籌碼 |
| 💰 交易 | Trading | 自動下單執行交易 | 即時交易 | 全部資料+帳戶狀態 |
| 🔧 函數 | Function | 封裝可重用的邏輯 | 被其他腳本調用 | 依調用環境 |

---

## 🎯 類型一：指標 (Indicator)

### 用途
在圖表上計算並顯示技術指標，幫助分析價格走勢。

### 特點
- **視覺化**：結果會繪製在圖表上
- **即時計算**：每根 K 棒更新時重新計算
- **多輸出**：可以輸出多條線或多個數值

### 可用資料
- ✅ OHLCV（開高低收量）
- ✅ 時間資訊（Date、Time）
- ✅ 基本財務數據（需額外設定）
- ❌ 帳戶資訊
- ❌ 持倉資訊

### 典型結構
```xscript
// 參數宣告
input: Length(14, "週期");
input: Price(Close, "計算價格");

// 變數宣告
vars: MyValue(0);

// 主要邏輯
MyValue = Average(Price, Length);

// 輸出繪圖
Plot1(MyValue, "指標線");
Plot2(MyValue * 1.1, "上軌");
Plot3(MyValue * 0.9, "下軌");
```

### 常見範例
- 移動平均線（MA、EMA）
- 相對強弱指標（RSI）
- 布林通道（Bollinger Bands）
- MACD
- KD 隨機指標

### 注意事項
1. **效能考量**：複雜計算會影響圖表載入速度
2. **繪圖限制**：最多同時繪製有限條線（通常8條）
3. **資料頻率**：注意 K 棒週期與計算邏輯的匹配
4. **向前參考**：不可使用未來資料（避免 look-ahead bias）

### 最佳實踐
- 使用 `input` 讓參數可調整
- 適當使用 `vars` 儲存中間計算結果
- 為每條輸出線設定清楚的名稱和顏色
- 避免過度複雜的計算

## 🔔 類型二：警示 (Alert)

### 用途
監控特定條件，條件滿足時發送通知給使用者。

### 特點
- **即時監控**：持續監控市場狀況
- **條件觸發**：只在條件滿足時動作
- **多檔監控**：可以同時監控多檔股票
- **通知方式**：彈窗、聲音、推播

### 可用資料
- ✅ OHLCV（開高低收量）
- ✅ 時間資訊
- ✅ 財務數據
- ✅ 技術指標
- ❌ 帳戶資訊
- ❌ 持倉資訊

### 典型結構
```xscript
// 參數宣告
input: RSI_Length(14, "RSI週期");
input: RSI_Threshold(70, "超買門檻");

// 變數宣告
vars: MyRSI(0);
vars: AlertTriggered(false);

// 計算指標
MyRSI = RSI(Close, RSI_Length);

// 條件判斷
if MyRSI crosses over RSI_Threshold and not AlertTriggered then begin
    // 設定警示
    Alert("RSI超買警示");
    AlertTriggered = true;
end;

// 重置機制
if MyRSI < RSI_Threshold - 5 then
    AlertTriggered = false;
```

### 常見範例
- 價格突破警示
- RSI 超買超賣警示
- 均線交叉警示
- 成交量異常警示
- Pin Bar 型態警示

### 注意事項
1. **避免重複觸發**：使用標記變數防止同一條件連續觸發
2. **重置機制**：設計合理的警示重置條件
3. **效能影響**：監控太多股票會影響系統效能
4. **誤報控制**：增加篩選條件降低假警示

### 最佳實踐
- 使用 `intrabarpersist` 在 K 棒內保持狀態
- 設計清晰的觸發和重置邏輯
- 提供有用的警示訊息（包含股票代碼、價格等）
- 使用多重條件過濾降低雜訊

## 🔍 類型三：選股 (Screener)

### 用途
從大量股票中篩選出符合特定條件的標的。

### 特點
- **批次處理**：一次掃描整個市場
- **條件篩選**：設定多重篩選條件
- **結果排序**：可依指定指標排序
- **定期執行**：通常每日或固定時間執行

### 可用資料
- ✅ OHLCV（開高低收量）
- ✅ 時間資訊
- ✅ 財務數據（營收、EPS、本益比等）
- ✅ 籌碼資訊（法人買賣、融資券等）
- ✅ 產業分類
- ❌ 帳戶資訊
- ❌ 持倉資訊

### 典型結構
```xscript
// 參數宣告
input: MinVolume(1000, "最小成交量(張)");
input: RSI_Lower(30, "RSI下限");
input: RSI_Upper(70, "RSI上限");

// 變數宣告
vars: MyRSI(0);
vars: VolumeInLots(0);
vars: Condition1(false);
vars: Condition2(false);

// 計算
MyRSI = RSI(Close, 14);
VolumeInLots = Volume / 1000;  // 轉換為張數

// 條件判斷
Condition1 = VolumeInLots > MinVolume;
Condition2 = MyRSI > RSI_Lower and MyRSI < RSI_Upper;

// 輸出結果
if Condition1 and Condition2 then
    Ret = 1;  // 符合條件
else
    Ret = 0;  // 不符合

// 可選：輸出數值供排序
OutputField(1, MyRSI);
OutputField(2, VolumeInLots);
```

### 常見範例
- 突破新高選股
- RSI 超賣選股
- 均線多頭排列選股
- 法人買超選股
- 產業龍頭選股

### 注意事項
1. **效能考量**：複雜計算會拖慢掃描速度
2. **資料時間點**：注意使用的資料是否為最新
3. **條件互斥**：避免設定互相矛盾的條件
4. **邊界情況**：處理資料缺失或異常值

### 最佳實踐
- 使用清晰的變數名稱表達條件
- 分步驟計算複雜條件
- 適當使用 `OutputField` 輸出關鍵數值
- 測試邊界情況和異常資料

## 💰 類型四：交易 (Trading/AutoTrade)

### 用途
根據策略自動執行買賣交易。

### 特點
- **自動下單**：系統自動執行交易
- **資金管理**：需要處理部位大小和風險
- **狀態追蹤**：追蹤持倉和訂單狀態
- **風險控制**：停損停利機制

### 可用資料
- ✅ OHLCV（開高低收量）
- ✅ 時間資訊
- ✅ 財務數據
- ✅ 技術指標
- ✅ 帳戶資訊（資金、可用保證金）
- ✅ 持倉資訊（部位、損益）
- ✅ 訂單狀態

### 典型結構
```xscript
// 參數宣告
input: FastMA(5, "快線週期");
input: SlowMA(20, "慢線週期");
input: StopLoss(2, "停損%");
input: TakeProfit(5, "停利%");
input: Lots(1, "每次交易張數");

// 變數宣告
vars: Fast(0), Slow(0);
vars: EntryPrice(0);

// 計算指標
Fast = Average(Close, FastMA);
Slow = Average(Close, SlowMA);

// 進場邏輯
if MarketPosition = 0 then begin  // 無部位
    if Fast crosses over Slow then begin
        Buy(Lots, "多單進場");
        EntryPrice = Close;
    end;
end;

// 出場邏輯
if MarketPosition = 1 then begin  // 持有多單
    // 停損
    if Close <= EntryPrice * (1 - StopLoss/100) then
        Sell(Lots, "停損出場");
    
    // 停利
    if Close >= EntryPrice * (1 + TakeProfit/100) then
        Sell(Lots, "停利出場");
    
    // 訊號反轉
    if Fast crosses under Slow then
        Sell(Lots, "訊號出場");
end;
```

### 常見範例
- 均線交叉策略
- 突破策略
- 網格交易
- 套利策略

### 注意事項
1. **滑價處理**：實際成交價可能與訊號價不同
2. **資金管理**：確保有足夠資金和保證金
3. **手續費成本**：計算交易成本對策略的影響
4. **市場限制**：注意漲跌停、暫停交易等狀況
5. **風險控制**：**必須**設定停損機制
6. **測試驗證**：充分回測和模擬交易

### 最佳實踐
- 明確的進場和出場邏輯
- 完善的風險控制機制
- 合理的資金管理
- 記錄交易原因（使用有意義的訂單名稱）
- 避免過度交易
- 充分的回測驗證

### ⚠️ 重要警告
交易腳本直接涉及真實資金，開發時務必：
1. 在模擬環境充分測試
2. 使用小資金驗證
3. 設定完善的風險控制
4. 持續監控執行狀況
5. 定期檢討和優化

## 🔧 類型五：函數 (Function)

### 用途
封裝可重用的邏輯，供其他腳本調用。

### 特點
- **模組化**：將複雜邏輯封裝成獨立單元
- **可重用**：多個腳本可以共用同一函數
- **參數化**：透過參數靈活使用
- **維護性**：集中管理邏輯，易於更新

### 可用資料
依調用它的腳本類型而定。

### 典型結構
```xscript
// 函數定義
inputs: Price(numericseries), Length(numericsimple);
variables: Sum(0), i(0);

// 函數邏輯
Sum = 0;
for i = 0 to Length - 1 begin
    Sum = Sum + Price[i];
end;

MyAverage = Sum / Length;  // 返回值

// 使用範例（在其他腳本中）
vars: MA10(0);
MA10 = MyAverage(Close, 10);
```

### 常見範例
- 自訂移動平均
- 自訂波動率計算
- 價格型態識別
- 支撐壓力計算
- 資金管理函數

### 注意事項
1. **參數類型**：明確定義參數類型（numericseries、numericsimple）
2. **返回值**：清楚定義返回值的意義
3. **效能影響**：避免過度複雜的計算
4. **依賴管理**：注意函數之間的依賴關係
5. **文檔維護**：**必須**在 `FUNCTIONS_INDEX.md` 登記

### 最佳實踐
- 單一職責：每個函數只做一件事
- 清晰命名：函數名稱表達其功能
- 參數驗證：檢查參數合理性
- 註解說明：說明函數用途和參數
- 測試驗證：獨立測試函數正確性
- **登記索引**：在 `scripts/functions/FUNCTIONS_INDEX.md` 登記

## 🎯 選擇指南

### 什麼時候使用指標？
- ✅ 需要視覺化分析工具
- ✅ 開發新的技術指標
- ✅ 研究價格型態
- ❌ 需要監控多檔股票
- ❌ 需要自動交易

### 什麼時候使用警示？
- ✅ 需要即時監控多檔股票
- ✅ 特定條件觸發通知
- ✅ 不需要自動交易
- ❌ 需要視覺化圖表
- ❌ 需要掃描全市場

### 什麼時候使用選股？
- ✅ 需要掃描全市場
- ✅ 根據財務、籌碼條件篩選
- ✅ 定期尋找交易機會
- ❌ 需要即時監控
- ❌ 需要自動交易

### 什麼時候使用交易腳本？
- ✅ 策略邏輯明確且經過驗證
- ✅ 需要自動執行交易
- ✅ 有完善的風險控制
- ❌ 策略尚未驗證
- ❌ 只需要提醒不需要交易

### 什麼時候使用函數？
- ✅ 邏輯會被多個腳本重用
- ✅ 複雜計算需要封裝
- ✅ 便於維護和更新
- ❌ 只使用一次的邏輯
- ❌ 簡單的內建函數可解決
