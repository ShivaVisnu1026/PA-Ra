# AI 提示詞指南

> 本指南教你如何與 AI 有效溝通，獲得最佳的 XScript 開發協助

---

## 🎯 核心原則

### 1. 明確性原則
AI 需要清楚的指令才能提供準確的協助。

**❌ 模糊的請求**：
```
幫我寫個警示
```

**✅ 明確的請求**：
```
幫我寫一個 XScript 警示腳本：
- 類型：警示腳本（Alert）
- 功能：當 RSI(14) 超過 70 時發出警示
- 需求：避免在同一根 K 棒內重複觸發
- 週期：5 分鐘線
```

### 2. 上下文原則
提供足夠的背景資訊讓 AI 理解情境。

**❌ 缺乏上下文**：
```
這段程式碼有問題
[貼上程式碼]
```

**✅ 完整上下文**：
```
我在開發一個選股腳本，想篩選出成交量大於 1000 張且 RSI 在 30-70 之間的股票。
以下是我的程式碼：
[貼上程式碼]

執行時出現錯誤：[錯誤訊息]

我已經嘗試過：[說明嘗試過的方法]
```

### 3. 具體性原則
具體的需求比抽象的概念更容易實現。

**❌ 抽象的需求**：
```
我想做個很賺錢的交易策略
```

**✅ 具體的需求**：
```
我想開發一個均線交叉策略：
- 進場：5MA 向上穿越 20MA
- 出場：5MA 向下穿越 20MA 或停損 2%
- 部位：每次 1 張
- 適用：台股期貨，15 分鐘線
```

```

**技術需求**：

- 條件：[觸發條件]
- 參數：[可調整的參數]
- 週期：[K 棒週期]

**特殊需求**：
- [任何特殊要求，例如避免重複觸發]

**參考資料**：
- [如有類似的範例，請提供路徑或連結]
```

**範例使用**：
```
我需要一個警示腳本

**功能描述**：
偵測 Pin Bar 型態並發出警示。Pin Bar 定義為下影線長度超過實體的 2 倍。

**技術需求**：
- 指標：無需額外指標，直接計算 OHLC
- 條件：當前 K 棒形成 Pin Bar
- 參數：
  - 影線比實體倍數（預設 2）
  - 最小實體大小（避免十字線）
- 週期：日線

**特殊需求**：
- 只在 K 棒收盤時檢查（避免 K 棒內重複觸發）
- 警示訊息要包含股票代碼和當前價格

**參考資料**：
- examples/signals/Pin_Bar_Reversal_警示.xs
```

## 💡 進階技巧

### 技巧 1：分步驟請求

對於複雜的需求，分多個步驟請求：

**步驟 1**：請求基本架構
```
請幫我設計一個均線交叉策略的基本架構，
包含參數定義、進場出場邏輯的框架即可，
先不需要詳細實現。
```

**步驟 2**：完善進場邏輯
```
基於以下架構，請幫我完善進場邏輯：
[貼上步驟1的結果]

進場條件：
- 快線向上穿越慢線
- 成交量大於 20 日均量
- RSI 在 30-70 之間
```

**步驟 3**：添加風險控制
```
基於以下程式碼，請添加停損停利機制：
[貼上步驟2的結果]

風控需求：
- 停損：-2%
- 停利：+5%
```

### 技巧 2：提供正反範例

說明什麼是對的，什麼是錯的：

```
我想寫一個函數來計算自訂均線，但不確定參數定義是否正確。

**我認為對的寫法**：
\`\`\`xscript
inputs: Price(numericseries), Length(numericsimple);
\`\`\`

**我不確定的部分**：
- Price 用 numericseries 對嗎？
- Length 用 numericsimple 還是 numericseries？
- 返回值該怎麼定義？
```

### 技巧 3：引用專案資源

明確引用專案中的資源：

```
請參考 examples/signals/Pin_Bar_Reversal_警示.xs，
幫我寫一個類似的腳本，但改成偵測長上影線（看跌 Pin Bar）。

主要差異：
- 原本偵測長下影線，改成長上影線
- 原本是看漲訊號，改成看跌訊號
```

### 技巧 4：指定回應格式

告訴 AI 你希望的回應方式：

```
請解釋 intrabarpersist 的使用，並提供：

1. 概念說明（2-3 句話）
2. 使用時機（列點說明）
3. 完整範例（帶註解的程式碼）
4. 常見錯誤（什麼情況不該用）
```

## 🚫 避免的提示詞模式

### ❌ 過於簡短

**不好**：
```
寫個 RSI 警示
```

**問題**：
- 沒有說明 RSI 參數
- 沒有說明警示條件
- 沒有說明特殊需求

### ❌ 過於模糊

**不好**：
```
我的程式碼有問題，幫我看看
[貼上 100 行程式碼]
```

**問題**：
- 沒有說明問題是什麼
- 沒有說明預期行為
- AI 需要猜測問題點

### ❌ 假設 AI 知道背景

**不好**：
```
像昨天那個腳本，但改成用 MACD
```

**問題**：
- AI 沒有「昨天」的記憶（每次對話是獨立的）
- 需要明確提供參考資料

### ❌ 要求不可能的功能

**不好**：
```
寫一個 100% 獲利的交易策略
```

**問題**：
- 不存在完美策略
- 應該聚焦在具體的技術實現

## 📚 XScript 特定技巧

### 說明腳本類型

每次請求時明確說明類型：

```
我需要一個【選股腳本】，用來篩選...
        ^^^^^^^
        明確類型
```

為什麼重要：
- 不同類型有不同的資料存取權限
- 不同類型有不同的典型結構
- 不同類型有不同的限制

參考：[五種腳本類型說明](SCRIPT_TYPES.md)

### 說明資料週期

明確說明使用的 K 棒週期：

```
這個警示要運行在【5分鐘線】上
                ^^^^^^^
```

為什麼重要：
- 影響是否需要 intrabarpersist
- 影響效能考量
- 影響資料計算方式

### 提及已查閱的文檔

如果已經查閱過文檔，請說明：

```
我已經看過 docs/learning/intrabarpersist學習.md，
理解了基本概念，但在實際使用時遇到...
```

好處：
- AI 不會重複解釋已知內容
- 可以提供更進階的協助

### 說明開發階段

說明腳本處於哪個階段：

```
這是一個還在測試的 brainstorm 腳本，主要想驗證...
```

或

```
這是準備放入 production 的腳本，請幫我檢查是否符合規範
```

好處：
- AI 知道要求的品質水準
- 提供適合階段的建議

## 📖 實際案例

### 案例 1：從零開始的完整請求

```
我需要開發一個警示腳本，以下是完整需求：

**腳本類型**：警示（Alert）

**功能描述**：
監控多檔股票，當出現「假突破」型態時發出警示。
假突破定義：價格突破 20 日高點，但在 3 根 K 棒內跌破突破點。

**技術需求**：
- 計算 20 日最高價
- 偵測價格突破
- 追蹤突破後的價格走勢
- 判斷是否為假突破

**參數設定**：
- 回溯期數：20（可調整）
- 確認 K 棒數：3（可調整）

**資料週期**：
日線（所以不需要 intrabarpersist）

**特殊需求**：
- 同一次突破只警示一次
- 如果是真突破（沒有跌破），之後再次突破新高時可以再次警示

**預期輸出**：
警示訊息包含：股票代碼、突破價格、當前價格

**參考資料**：
可以參考 examples/ 中的警示腳本結構

請提供完整的程式碼和說明。
```

### 案例 2：具體的除錯請求

```
**問題**：選股腳本執行時很慢

**腳本類型**：選股（Screener）

**程式碼**：
\`\`\`xscript
vars: i(0), count(0);
vars: vol_sum(0), vol_avg(0);

// 計算 20 日平均量
count = 0;
vol_sum = 0;
for i = 0 to 19 begin
    vol_sum = vol_sum + Volume[i];
    count = count + 1;
end;
vol_avg = vol_sum / count;

// 檢查條件
if Volume > vol_avg * 1.5 then
    Ret = 1
else
    Ret = 0;
\`\`\`

**問題分析**：
這個腳本要掃描整個市場（約 2000 檔股票），執行時間超過 5 分鐘。

**已嘗試**：
1. 檢查邏輯沒有錯誤
2. 簡化條件後還是很慢

**請求**：
請幫我優化這段程式碼，提升執行效能。如果有內建函數可以取代迴圈計算，請告訴我。
```

## 🎯 總結

### 好的提示詞的特徵
1. ✅ **明確**：清楚說明要什麼
2. ✅ **完整**：提供足夠的上下文
3. ✅ **具體**：技術細節明確
4. ✅ **結構化**：使用標題、列表組織資訊
5. ✅ **引用資源**：提及相關文檔和範例

### 與 AI 協作的心態
- 把 AI 當作配對編程夥伴
- 提供清楚的需求和回饋
- 不要害怕請求解釋和澄清
- 分步驟處理複雜任務
- 持續學習和改進

### 記住
> 清晰的提示詞 = 準確的協助

花時間寫好提示詞，會節省後續來回溝通的時間！
