## 簡介

這是 TradingView 上廣受歡迎的 "Price Action All In One" 指標的 XQScript 移植版本。該指標整合了多個價格行為分析功能，適合遵循 Al Brooks 價格行為交易法則的交易者。

### 原始指標功能

1. **Gap Series（跳空系列）**
   - Traditional Gap（傳統跳空）
   - Tail Gap（影線跳空）
   - Body Gap（實體跳空）

2. **Bar Count（K棒計數）**
   - 不同時間週期的K棒編號

3. **Multiple EMAs（多週期移動平均）**
   - 支援跨時間週期EMA

4. **Previous Values（前值水平線）**
   - 前日/週/月 的高低點、收盤價等

5. **IO Pattern Matching（內外包型態）**
   - II Pattern（Inside Inside）
   - OO Pattern（Outside Outside）
   - IOI Pattern（Inside-Outside-Inside）

6. **H & L Count（高低點計數）**
   - H1, H2, H3...（多頭推進）
   - L1, L2, L3...（空頭推進）

## 檔案說明

### 1. `PriceAction_AllInOne_Indicator.xs`
**類型**：指標腳本（Indicator）

**功能**：
- ✅ 繪製多週期 EMA
- ✅ 顯示前值水平線（前日/週高低點等）
- ✅ 偵測跳空（傳統、影線、實體）
- ✅ 偵測 IO 型態（II、OO、IOI）
- ✅ 輸出偵測結果到欄位

**使用時機**：
- 作為技術分析主要指標
- 需要在圖表上顯示 EMA 和前值水平線
- 需要監控多個商品的型態狀態

### 2. `PriceAction_AllInOne_Alert.xs`
**類型**：警示腳本（Alert/Sensor）

**功能**：
- ✅ 跳空即時警示
- ✅ IO 型態即時警示
- ✅ H/L 計數追蹤與警示
- ✅ 自訂警示訊息

**使用時機**：
- 需要即時通知特定型態出現
- 監控特定商品的關鍵價格行為
- 搭配手機 App 接收推播通知

## 安裝使用

### 步驟 1：匯入腳本

1. 開啟 XQ 全球贏家軟體
2. 進入「腳本編輯器」
3. 新增腳本，複製貼上對應的 `.xs` 檔案內容
4. 儲存腳本

### 步驟 2：使用指標

#### 方法 A：加入技術指標
```
1. 開啟商品K線圖
2. 右鍵 → 加入技術指標
3. 選擇 "PriceAction_AllInOne_Indicator"
4. 調整參數（可選）
```

#### 方法 B：設定警示
```
1. 右鍵商品 → 加入警示
2. 選擇 "PriceAction_AllInOne_Alert"
3. 設定觸發條件
4. 開啟通知
```

#### EMA 設定
| 參數 | 預設值 | 說明 |
|------|--------|------|
| `ema1stUse` | true | 是否使用第1條EMA |
| `ema1stLength` | 20 | EMA1 週期 |
| `ema1stTF` | "" | EMA1 時間週期（空白=當前週期） |
| `ema2ndUse` | true | 是否使用第2條EMA |
| `ema2ndLength` | 20 | EMA2 週期 |
| `ema2ndTF` | "15" | EMA2 時間週期（15分鐘） |
| `ema3rdUse` | true | 是否使用第3條EMA |
| `ema3rdLength` | 20 | EMA3 週期 |
| `ema3rdTF` | "60" | EMA3 時間週期（60分鐘） |

**時間週期代碼**：
- 分鐘：直接輸入數字（如 "5", "15", "60"）
- 日線：`"D"`
- 週線：`"W"`
- 月線：`"M"`

| 參數 | 預設值 | 說明 |

| `pv1stUse` | true | 是否使用前值1 |

| `pv2ndSource` | "high" | 前值2來源 |

| `pv3rdUse` | true | 是否使用前值3 |
| `pv3rdSource` | "low" | 前值3來源 |
| `pv3rdTF` | "D" | 前值3週期 |

## 使用建議

### 📈 日內交易（5分鐘K線）

**建議設定**：
```
指標版：
- EMA1: 20期, 當前週期（5分鐘）
- EMA2: 20期, 15分鐘週期
- EMA3: 20期, 60分鐘週期
- 前值1: 前日收盤
- 前值2: 前日高點
- 前值3: 前日低點

警示版：
- 開啟傳統跳空警示
- 開啟 IOI 型態警示（突破型態）
- 開啟 H/L 計數（追蹤推進）
```

**應用場景**：
1. **趨勢判斷**：觀察價格與多週期 EMA 的關係
2. **支撐壓力**：利用前日高低點作為關鍵價位
3. **突破確認**：IOI 型態出現時留意突破方向
4. **推進力度**：H/L 計數追蹤趨勢強度

### 📊 波段交易（日線）

**建議設定**：
```
指標版：
- EMA1: 8期, 日線
- EMA2: 20期, 日線
- EMA3: 60期, 日線
- 前值1: 前週收盤（W）
- 前值2: 前週高點（W）
- 前值3: 前週低點（W）

警示版：
- 開啟傳統跳空警示
- 開啟 OO 型態警示（擴張型態）
- 關閉 H/L 計數（日線上較不實用）
```

### 🎯 關鍵概念

#### 跳空（Gaps）
- **傳統跳空**：K棒之間有明顯空隙，代表強烈的單向力量
- **影線跳空**：更常見，影線之間的跳空
- **實體跳空**：實體之間的跳空，力道最強

**交易意義**：
- 向上跳空：買方強勢，可能續漲
- 向下跳空：賣方強勢，可能續跌
- 連續跳空：趨勢加速的徵兆
- 跳空回補：可能的反轉訊號

#### IO 型態（Inside/Outside Patterns）
- **II（Inside Inside）**：連續內包，代表盤整收斂，即將突破
- **OO（Outside Outside）**：連續外包，代表波動擴大
- **IOI（Inside-Outside-Inside）**：內外內，強烈的突破型態

**交易意義**：
- II 型態：等待突破方向確認後進場
- OO 型態：波動擴大，注意風險
- IOI 型態：突破確認，可作為進場訊號

#### H/L 計數
- **H1, H2, H3...**：多頭每次創新高的標記
- **L1, L2, L3...**：空頭每次創新低的標記

**交易意義**：
- H 計數增加：多頭推進有力
- L 計數增加：空頭推進有力
- 計數重置：推進失敗，可能反轉

## 限制說明

### 技術限制

1. **無法繪製跳空區域**
   - 原因：XQScript 不支援 Box 繪圖
   - 替代：使用警示通知 + OutputField 顯示

2. **無法在K棒上標註文字**
   - 原因：XQScript 不支援動態 Label
   - 替代：使用 OutputField 或警示訊息

3. **跨時間週期功能受限**
   - 原因：XQScript 的跨週期函數較複雜
   - 替代：使用 GetField() 取得跨週期資料

4. **K棒計數功能未實現**
   - 原因：無法在每根K棒上顯示編號
   - 說明：此功能在 XQScript 中無實用替代方案

### 使用限制

1. **指標繪製數量限制**
   - XQScript 指標最多 999 條 Plot
   - 本指標使用 6 條（3條EMA + 3條前值）

2. **警示觸發限制**
   - 每個警示腳本每根K棒只能觸發一次
   - 建議分開設定不同類型的警示

3. **回測資料限制**
   - `settotalbar(250)` 設定可用K棒數
   - 計算複雜指標時可能需要更多歷史資料

## 常見問題

### Q1：為什麼 EMA 沒有顯示？
**A**：檢查以下項目：
1. 確認對應的 `emaXstUse` 參數為 `true`
2. 檢查時間週期設定是否正確
3. 確認圖表有足夠的歷史資料

### Q2：為什麼跨時間週期 EMA 顯示不正確？
**A**：XQScript 的跨週期功能有限制：
- 當前版本使用簡化的跨週期邏輯
- 建議在對應的時間週期圖表上使用指標
- 例如：使用 15 分鐘 EMA 時，建議在 5 分鐘圖上觀察

### Q3：前值水平線為什麼是階梯狀？
**A**：這是正常的：
- `GetField()[1]` 取得前一個完整週期的值
- 值會在週期轉換時更新
- 如果需要平滑的線，可調整邏輯使用當前值

### Q4：警示為什麼沒有觸發？
**A**：檢查以下項目：
1. 確認對應的 `alertXXX` 參數為 `true`
2. 檢查是否設定了警示條件
3. 確認商品有足夠的波動產生訊號
4. 某些型態（如IOI）需要特定K棒組合才會觸發

### Q5：OutputField 顯示的數字代表什麼？
**A**：
- **傳統跳空**：1=向上, -1=向下, 0=無
- **IO型態**：1=OO, 2=II, 3=IOI, 0=無
- **H/L計數**：當前的 H 或 L 計數值

### Q6：可以同時使用指標版和警示版嗎？
**A**：可以，建議的組合：
- 圖表上：使用指標版（視覺化 EMA 和前值）
- 警示清單：使用警示版（即時通知型態）

### Q7：如何調整 H/L 計數的敏感度？
**A**：調整 `hlInterval` 參數：
- 增大（如 1.5, 2.0）：降低敏感度，減少訊號
- 減小（如 0.5）：提高敏感度，增加訊號
- 預設 1.0 適用於大多數情況
